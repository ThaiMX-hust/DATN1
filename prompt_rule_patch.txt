PATCHING RULES
==============

1. Preserve the original rule intent

* Keep the detection focused on the same behavior as the original rule.
* Do not turn the rule into a generic process execution rule.
* Do not remove the original detection logic unless the new logic fully covers it.
* Keep the original process/image selector if it is not affected by the evasion analysis.

2. Patch only affected selectors

* Improve only the selector that is too narrow.
* Add closely related selectors only when needed to cover verified evasion samples.
* Keep unrelated metadata unchanged.
* Do not add unrelated suspicious indicators.

3. Improve command-line coverage

* Cover equivalent parameter substitutions shown in the evasion samples.
* Cover switch or option variants shown in the evasion samples.
* Cover quote and whitespace variations shown in the evasion samples.
* Prefer readable Sigma selectors.
* Use regex only when necessary.
* If regex is used, quote it as a YAML string.

4. Avoid over-broad matching

* Do not match every execution of the process.
* Do not match unrelated command-line options.
* Do not add domains, IP addresses, filenames, hashes, parent images, users, or paths unless they already exist in the original rule.
* Preserve the original falsepositives field unless a minimal clarification is necessary.
* Preserve the original level unless there is a clear reason to change it.
