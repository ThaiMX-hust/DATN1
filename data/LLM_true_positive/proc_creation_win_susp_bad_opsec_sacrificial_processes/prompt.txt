You are a SIEM detection engineering assistant with expert knowledge of Sigma rules,
Windows process creation logs, and command-line parsing.

Task:
Generate realistic, benign command-line examples that would produce true-positive
Windows process creation events for the Sigma rule below. The commands are for
authorized lab validation and regression testing only.

Requirements:
- Output command lines only, one per line. Do not include explanations, numbering,
  bullets, code fences, or any extra text.
- Each command must meaningfully match at least one detection path in the Sigma rule.
- Prefer safe placeholders and harmless actions, such as TEST-HOST, example.com,
  192.0.2.10, C:\Temp\test.txt, echo, whoami, dir, or temporary test files.
- Do not provide commands that steal credentials, disable security tools, establish
  persistence, evade detection, exfiltrate data, or cause damage.
- Include variations only when they exercise materially different rule conditions,
  such as image path, flag order, quoting, casing, argument aliases, or suspicious
  path choices.
- If a binary is not available by default on Windows, assume it is installed at the
  exact path required by the Sigma rule detection logic.
- If no safe command can be produced, output a single line: NO_SAFE_COMMAND

Sigma_rule:
title: Bad Opsec Defaults Sacrificial Processes With Improper Arguments
id: a7c3d773-caef-227e-a7e7-c2f13c622329
related:
    - id: f5647edc-a7bf-4737-ab50-ef8c60dc3add
      type: obsolete
status: test
description: |
    Detects attackers using tooling with bad opsec defaults.
    E.g. spawning a sacrificial process to inject a capability into the process without taking into account how the process is normally run.
    One trivial example of this is using rundll32.exe without arguments as a sacrificial process (default in CS, now highlighted by c2lint), running WerFault without arguments (Kraken - credit am0nsec), and other examples.
references:

    - https://www.cobaltstrike.com/help-opsec
    - https://twitter.com/CyberRaiju/status/1251492025678983169
    - https://learn.microsoft.com/en-us/windows-server/administration/windows-commands/regsvr32
    - https://learn.microsoft.com/en-us/windows-server/administration/windows-commands/rundll32
    - https://learn.microsoft.com/en-us/dotnet/framework/tools/regasm-exe-assembly-registration-tool
    - https://learn.microsoft.com/en-us/dotnet/framework/tools/regsvcs-exe-net-services-installation-tool
author: Oleg Kolesnikov @securonix invrep_de, oscd.community, Florian Roth (Nextron Systems), Christian Burkard (Nextron Systems)
